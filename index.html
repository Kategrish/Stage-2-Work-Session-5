Important concepts

Lesson 1: The Basics of the Web and HTML

World Wide Web

The World Wide Web is a huge collection of HTML documents. It was invented in the 1990s and now it contains over 30 billion web pages.

This is how it works. When you visit a web page your computer sends a HTTP request to a server via the Internet. The server finds the HTML document with all the files for the page and sends it back to your computer. The browser on your computer interprets the information sent by the server and shows you the web page. Here is a good video about the components of the web.

HTML

HTML (Hypertext Markup Language) is a special code for computers. HTML documents contain text content (what you actually see on the page) and markup (how the content is arranged).

Tags

HTML markup is made of different tags. Together with content they make up different types of elements. Here is a video about structure of HTML documents.

Tags are essential and you should be very carefull when writing HTML because computers are "stupid". Tags are instructions for them so even a smallest mistake can lead to a big problem.

Inline and block elements

HTML elements can be inline or block. Block elements unlike inline ones create an "invisible box" around the content. The concept of boxes is very important in web programming.

Lesson 2: Creating a Structured Document with HTML

Developer Tools

Developer tools in Chrome browser shows the structure of the page. In Developer tools it is clearly seen that HTML elements form boxes on the page.

Page Structure

The structure of any page is "tree-like". It means HTML elements can be inside other elements.

If you look at the structure of the page in Developer tools or any other tools you can see sideway triangles next to some of the elements. They indicate the "branching" structure and that these elements contain other elements. The more deeply "nested" in the structure an element is, the more indented it is in the code.

"Boxifying" design

"Everything on the Web is boxes". Before writing a code of a web page it is important to "boxify" the design first - to cut your future page into boxes.

Lesson 3: Adding CSS Style to HTML Structure

CSS

CSS (Cascading Style Sheets) is a language which allows programmers apply style to HTML elements. It also helps them avoid repetition.

Avoiding repetition

"Don't repeat yourself" (DRY) is an important principle for software developers. And if you want to avoid repetition in your code you have to plan it carefully. First of all, it will save your time: instead of applying attributes to every element in HTML code you can distinguish similar elements and apply certain attributes to them all in CSS. Secondly, it will help to avoid mistakes and bugs: repetition makes a code very complex and that can lead to errors, especially if the developer decides to change something in the code. So CSS is a great tool which makes a developer's life much easier.

Important concepts - Telling Computers What to Do

Lesson 1: Introduction to 'Serious' Programming

Computers and Computer Programming

Computers are just machines. But unlike other machines like toasters or fridges, which are designed to do only a few things, computers can do anything. And they can do so with the help of programs, which tells them what to do. Without being programmed computers can do nothing so a program has to be a sequence of steps with instructions for a computer to do something useful.

Programming Languages

For organizing program instructions we need special programming languages. And like human languages programming languages have grammar too. But unlike human languages grammar grammar in programming is very strict and lacks any ambiguity. In languages like English, people can understand sentences that aren't grammatically correct or ambiguous sentences depending on the situation. Computers can't. If you make even a smallest mistake in the syntax, your code will not run.

Python

Python is a programming language. It also has a very strict grammar. For example, an expression like 2 + 2 can be executed by Python, but 2 + 2 + can't because it's not part of the language.

Lesson 2: Variables and Strings

Variables in Python

Programmers use variables to give names to values. Variables store the value of important data. They also make programs easier to understand. For example, looking at the code print 299792458 * 1.0 / 1000000000 * 100 it is hard to tell what 299792458 is and why these numbers are multiplied. But if we say that 299792458 is the speed of light and the code calculates how far light travels in centimeters in one nanosecond everything gets much clearer.

Assignment

So to tell Python that 299792458 is the speed of light we need to assign a variable to this value. We can do so by writing speed_of_light = 299792458. The equals sign in this code 'takes the value of' not 'is the same as' like in maths. We can reassign a variable to a different value later in our code. Moreover, variables can be used in their own assignment expressions: for example, days = days + 1.

Strings

A string in Python is a sequence of characters surrounded by quotes. It's important to remember how strings behave in Python. For example, if we run print 4 + 4 the output will be 8. But the code '4' + '4' will give 44. Plus operator concatenates strings.

Lesson 3: Input -> Function -> Output

Functions

Functions, or procedures, are like small programs in code that are aimed at specific tasks. When given parameters they run and return a value to the main program. Functions are really useful because they help to divide our code into blocks, which can be developed independently. Functions once defined can be reused again and again.

Making and using functions

To make a function we need to start a line with a keyword def and then give the function a name followed by the function parameters in parentheses and a colon. Parameters are the values you pass to the function when you use (call) it later in your code. We also have to write a block ('body') of the function where we specify what to do with the input parameters. For example, the following code returns the next number:
def next_number(x):
answer = x + 1
return answer
 After you have defined a function, you can use it (call it) at any time, in any place of your code. To use a function we write the name of the function followed by the value or values in parentheses:
print next_number(45)
>>>46
 The return keyword tells Python what the output must be. Without it the output will be:
>>>None
